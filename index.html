Test-Web-1 API Documentations

Usage

  • /users endpoint:

    1. GET http://localhost:9000/users

      Request body:

      null

      Result:

      Returns all users registered in the website

    2. POST http://localhost:9000/users

      Request body:

      { "userName": "Insert username here" }

      Creates new user for the website

    3. PUT http://localhost:9000/users/:id

      Request body:

      { "userName: "insert changed username" }

      Set a new username for a specific userId

    4. DELETE http://localhost:9000/users/:id

      Request body:

      null

      Deletes a specific user based on userId

  • /events endpoint:

    1. GET http://localhost:9000/events

      Request body:

      null

      Returns all events in the website

    2. GET http://localhost:9000/events/:id

      Request body:

      null

      Returns a specific event in the website based on the eventId

    3. POST http://localhost:9000/events

      Request body:

      { "eventName": "Insert event name here", "eventDesc": "Insert event description here", "eventImage": "Insert event image URL here" }

      Creates new event for the website

    4. PUT http://localhost:9000/events/:id

      Request body:

      { "eventName": "Insert event name here", "eventDesc": "Insert event description here", "eventImage": "Insert event image URL here" }

      Update properties of an event.

    5. DELETE http://localhost:9000/events/:id

      Request body:

      null

      Deletes a specific event based on eventId

  • /events/:id/users endpoint:

    1. GET http://localhost:9000/events/:id/users

      Request body:

      null

      Returns all users attending a specific event.

    2. POST http://localhost:9000/events/:id/users

      Request body:

      { "user_id": "Insert userId here" }

      Add user to an event's attendance list

    3. DELETE http://localhost:9000/events/:id/users

      Request body:

      { "user_id": "Insert userId here" }

      Remove a user from a specific event
